# .worktreeinclude 自動生成 Hook セットアップ

## 概要

`generate-worktreeinclude.sh` は、リポジトリ内の `.env*` ファイルを検出し `.worktreeinclude` を自動生成するスクリプト。
Claude Code の hooks 機能（`settings.json`）に登録することで、`git worktree add` の実行前に自動で `.worktreeinclude` を生成できる。

## Claude Code Hook 登録方法

`~/.claude/settings.json`（またはプロジェクトの `.claude/settings.json`）に以下を追加:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "if echo \"$TOOL_INPUT\" | grep -q 'git worktree add'; then $SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh; fi"
          }
        ]
      }
    ]
  }
}
```

### 動作フロー

1. Claude Code が `Bash` ツールで `git worktree add` を実行しようとする
2. PreToolUse hook がトリガーされる
3. `generate-worktreeinclude.sh` がリポジトリルートに `.worktreeinclude` が存在するか確認
4. 存在しなければ `.env*` ファイルを検出してパターンを `.worktreeinclude` に書き出す
5. Claude Code が `git worktree add` を実行 → `.worktreeinclude` のパターンに基づき `.env*` ファイルが自動コピーされる

## スクリプト単体実行

```bash
# 基本実行（リポジトリルートで）
$SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh

# 既存ファイルを上書き
$SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh --force

# ドライラン（書き出し内容を確認）
$SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh --dry-run

# リポジトリルートを明示
$SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh --repo-root /path/to/repo
```

## 生成されるファイルの例

```
# Auto-generated by generate-worktreeinclude.sh
# Patterns for .env* files to be copied into new worktrees
# Regenerate: $SKILLS_DIR/_lib/scripts/generate-worktreeinclude.sh --force
.env
.env.local
**/.env
**/.env.example
**/.env.local
```

## 注意事項

- `.envrc`（direnv 用）は意図的に除外される
- `node_modules/`, `.git/`, `*-worktrees/` 配下は検索対象外
- `.worktreeinclude` が既に存在する場合はスキップ（`--force` で上書き可能）
- settings.json はリポジトリに含めない（ユーザーが手動で設定）
