# Trends Analyzer リファレンス

## トレンドスコアの解釈

### trend_direction（トレンド方向）

| 値 | 意味 | 判定基準 | SEO示唆 |
|----|------|----------|---------|
| rising | 上昇トレンド | 直近1/4期間の平均が全体平均より15%以上高い | 今すぐ記事化すべき。競合が少ないうちに上位獲得 |
| stable | 安定 | 変動が±15%以内 | 安定需要あり。質の高い記事で長期流入を狙える |
| declining | 下降トレンド | 直近1/4期間の平均が全体平均より15%以上低い | 優先度低。既存記事の更新は有効だが新規は非推奨 |

### avg_interest（平均関心度）

- **0-20**: ニッチ。競合は少ないがボリュームも小さい
- **20-50**: ミドルレンジ。狙い目のスイートスポット
- **50-80**: 高関心。競合も多いが流入期待大
- **80-100**: トップトレンド。大手メディアとの競合注意

### rising_queries（上昇クエリ）

関連する上昇中の検索クエリ。記事の切り口やロングテールキーワードとして活用。

## pytrends パラメータ

### timeframe 指定例

| 値 | 説明 |
|----|------|
| `today 1-m` | 直近1ヶ月 |
| `today 3-m` | 直近3ヶ月（デフォルト） |
| `today 12-m` | 直近12ヶ月 |
| `2024-01-01 2024-12-31` | カスタム期間 |

### geo リージョン例

| 値 | 説明 |
|----|------|
| `JP` | 日本（デフォルト） |
| `US` | アメリカ |
| `` (空) | グローバル |

## トラブルシューティング

### 429 Too Many Requests

pytrends はGoogle非公式APIのため、短時間に多数のリクエストを送るとレート制限される。

- スクリプトは自動リトライ（exponential backoff: 20s→40s→80s）を実装済み
- `--cache-dir` でキャッシュを有効にすると同一クエリの再実行を防げる
- キャッシュは24時間有効

### キーワード数の制限

- pytrends は1回のリクエストで最大5キーワードまで比較可能
- スクリプトは自動で5件ずつバッチ処理する
- バッチ間に5秒の待機時間を設定済み
