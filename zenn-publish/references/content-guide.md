# Zenn Content Guide

Zenn向けクロスポスト記事の書き方・テンプレート・変換ルール。

**差別化方針は `$SKILLS_DIR/blog-cross-post/references/cross-post-strategy.md` に従う。**

## Zenn読者の特性

| 項目 | 特徴 |
|------|------|
| 読者層 | モダン技術志向、個人開発者文化 |
| 好む切り口 | **Deep How + 意見**（実装 + 判断 + 持論） |
| 記事スタイル | 課題共感→実装+判断理由→持論、書き手の体験として語る |
| トーン | 個人の体験として語る（企業感を抑える） |
| フッター | 企業紹介なし（個人ブログ文化、企業感は嫌われる） |

## Zenn記事の設計思想

Zennで伸びる記事は「コードを貼って終わり」ではなく、**書き手の判断・意見・体験**が入ったディープダイブ記事。
薄いHow-toはZenn上に既に溢れており、差別化できない。

**3要素:**
1. **課題の共感** — 読者が「あるある」と感じる導入
2. **実装 + 判断理由** — コードだけでなく「なぜこう書くか」を添える
3. **持論・トレードオフ** — 他の方法との比較、書き手としての意見

**やってはいけないこと:**
- TL;DRで始める（好奇心を殺す）
- コードだけ貼って解説なし
- 「さらに深掘りしたい方へ」という宣伝感のある見出し
- playpark企業紹介セクション

## テンプレート

### tech-tips — Deep How + 意見

**切り口: 課題共感 → 実装 + 判断理由 → 持論・トレードオフ**

```markdown
---
title: "{課題・体験ベースのタイトル}"
emoji: "🛠️"
type: "tech"
topics: ["{topic1}", "{topic2}", "{topic3}"]
published: true
---

## {読者の課題を問いかける見出し}

{読者が「あるある」と感じる課題・場面の描写 - 150-200字}
具体的な痛みを2-3個箇条書きで示す。

## {解決アプローチの見出し（結論を含める）}

{アプローチの概要 + なぜこれを選んだか - 100-150字}

```{lang}
{CODE_EXAMPLE: 動作する完全なコード}
```

### なぜこう書くか

{コードの判断理由 - 200-300字}
「こう書いても動くが、こう書いた理由は〜」という形で、
代替案との比較を交えて判断プロセスを共有する。

| 方法 | メリット | デメリット |
|-----|---------|-----------|
| {採用した方法} | {メリット} | {デメリット} |
| {代替案} | {メリット} | {デメリット} |

## {実践で気づいたこと・落とし穴}

{実際に使ってみて分かったこと - 150-200字}
ドキュメントには書いていない、体験ベースの知見を共有する。

## まとめ: 自分はこう使っている

{持論・判断基準のまとめ - 100-150字}
「万人向けの正解」ではなく「自分はこう判断した」というスタンスで締める。

---

## 元記事

この記事は **[{ARTICLE_TITLE}]({ORIGINAL_URL}?utm_source=zenn&utm_medium=crosspost&utm_campaign={slug})** のコア部分を再構成したものです。元記事ではさらに：

- {DEPTH_1: 複数パターンの比較・使い分け}
- {DEPTH_2: エッジケースと対処法}
- {DEPTH_3: パフォーマンス最適化のコツ}

を扱っています。
```

### lab-reports — 課題共感 + 検証 + 持論

**切り口: 「期待と現実のギャップ」から始まり、検証結果 + 解釈 + 持論で締める**

```markdown
---
title: "{検証テーマ}を{期間/条件}試して分かった現実"
emoji: "🔬"
type: "tech"
topics: ["{topic1}", "{topic2}", "{topic3}"]
published: true
---

## {期待と現実のギャップを示す見出し}

{読者が期待しがちなこと + 実際にはそう簡単ではないという問題提起 - 150-200字}

## 何を検証したか

{検証の目的と条件を簡潔に - 100-150字}

## 結果: 期待どおりだったこと・そうでなかったこと

{検証結果をフラットに共有 - 200-300字}

| 検証項目 | 期待 | 実際 | 所感 |
|---------|------|------|------|
| {項目1} | {期待} | {結果} | {一言コメント} |
| {項目2} | {期待} | {結果} | {一言コメント} |

### 検証コード

```{lang}
{CODE_EXAMPLE: 検証に使用したコード}
```

## {意外だったこと / ハマったこと}

{ドキュメントには書いていない、実際にやって初めて分かったこと - 150-200字}
ここが記事の独自価値になる。

## 自分はこう使うことにした

{検証結果を踏まえた持論・判断 - 150-200字}
「万人向けの結論」ではなく「自分のプロジェクトではこう判断した」というスタンス。

---

## 元記事

この記事は **[{ARTICLE_TITLE}]({ORIGINAL_URL}?utm_source=zenn&utm_medium=crosspost&utm_campaign={slug})** のコア部分を再構成したものです。元記事ではさらに：

- {DEPTH_1: 詳細な検証環境・条件}
- {DEPTH_2: 生データと詳細な分析}
- {DEPTH_3: 失敗した試みと学び}

を扱っています。
```

## 導線スタイル

- 「さらに深掘りしたい方へ」は使わない（宣伝感が強い）
- 「元記事」というシンプルな見出しで、フラットにリンクする
- 企業紹介セクションは入れない

## Frontmatter変換ルール

```yaml
---
title: "記事タイトル"
emoji: "🚀"
type: "tech"
topics: ["nextjs", "react", "typescript"]
published: false
---
```

| Field | Source | Rule |
|-------|--------|------|
| title | title | 課題・体験ベースに再構成 |
| emoji | category | カテゴリから自動選択 |
| type | category | tech-tips/lab-reports → tech, その他 → idea |
| topics | tags | 小文字化、最大5個 |
| published | - | false（下書き投稿、確認後に手動でtrueに変更） |

> **Note**: Zenn は `canonical_url` を frontmatter でサポートしていない（[Issue #78](https://github.com/zenn-dev/zenn-community/issues/78) 未実装）。記事末尾のテキストリンクが唯一の手段。

### Emoji Mapping

| Category | Options |
|----------|---------|
| tech-tips | 🛠️ 💡 ⚙️ 🔧 |
| lab-reports | 🧪 🔬 🚀 ⚡ |

### Topics変換

```
# Original → Zenn
Next.js    → nextjs
React 19   → react
TypeScript → typescript
フロントエンド → frontend
```

ルール: 小文字化、スペース削除、バージョン番号削除、日本語→英語、最大5個

### Canonical URL

記事末尾に追加:

```markdown
---

**この記事は [playpark Blog](https://www.playpark.co.jp/blog/{slug}?utm_source=zenn&utm_medium=crosspost&utm_campaign={slug}) からの転載です。**
```

## SEO

### タイトル最適化

```
MDXブログを自作して踏んだ3つの地雷 — ISR・予約投稿・OGPの落とし穴
```

- 課題・体験ベース（「〜で踏んだ」「〜を試して分かった」「〜の現実」）
- 【】は使ってもよいが、ツール名より課題/体験を前面に
- 60文字以内推奨
- 読者の好奇心を引く表現（「地雷」「落とし穴」「現実」「意外だった」）

### Zenn固有SEO

- **Topics**: 検索に大きく影響、適切な5つを選択
- **Emoji**: 目立つ絵文字で CTR 向上
- **Publication**: 企業・チームなら Publication 活用

### キーワード傾向

| ユーザー傾向 | キーワード例 |
|-------------|-------------|
| 最新技術、実装詳細 | Next.js 16, App Router, RSC |

### 文字数ガイドライン

| セクション | 目安 |
|-----------|------|
| 課題の共感 | 150-200字 |
| 本文（コード + 判断理由） | 800-1200字 |
| まとめ・持論 | 100-150字 |
| 元記事リンク | 100-150字 |
| **合計** | **1400-2200字** |

### 転載後チェックリスト

- [ ] 記事末尾に転載元テキストリンク明記済み
- [ ] `published: false` で下書き投稿、表示確認後に公開
- [ ] タグ/トピックス適切に設定（5個以内、小文字）
- [ ] 企業紹介セクションが入っていないこと
