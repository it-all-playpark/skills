## Frontmatter変換ルール

```yaml
---
title: "記事タイトル"
tags:
  - name: Next.js
    version: "16"
  - name: React
private: false
---
```

| Field | Source | Rule |
|-------|--------|------|
| title | title | Why型に再構成（「なぜ〜」） |
| tags | tags | タグ名保持、バージョン分離（可能なら） |
| private | - | false |

### Tags変換

```yaml
# Original → Qiita
- name: Next.js
  version: "16"
- name: React
- name: TypeScript
- name: MDX
```

ルール: 元のタグ名を保持、バージョン情報があれば分離、最大5個

### Canonical URL

記事**冒頭**に追加:

```markdown
> この記事は [playpark Blog](https://www.playpark.co.jp/blog/{slug}) からの転載です。

---
```

## SEO

### タイトル最適化

```
【Next.js 16】MDXブログを自作した話 - ISRと予約投稿で「ちょうどいい」を目指す
```

- Why型（「なぜ〜」「〜を選んだ理由」）
- 【】でカテゴリ/技術を明示
- タグでキーワード補完

### Qiita固有SEO

- **Tags**: バージョン情報付きで検索精度向上
- **LGTM/ストック**: 初期の反応が検索順位に影響
- **限定共有 → 公開**: 下書き期間で内容精査

### キーワード傾向

| ユーザー傾向 | キーワード例 |
|-------------|-------------|
| 入門、ハウツー | 〇〇のやり方, 〇〇入門, 初心者 |

### 文字数ガイドライン

| セクション | 目安 |
|-----------|------|
| 学べること | 100-150字 |
| 背景・課題 | 200-300字 |
| 本文（解説中心） | 600-1000字 |
| まとめ | 100-150字 |
| 深掘り導線 | 150-200字 |
| **合計** | **1200-2000字** |
